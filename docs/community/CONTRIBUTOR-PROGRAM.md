# Contributor Recognition Program

> We believe that every contribution — a bug fix, a new module, a polished theme, or clearer docs — makes Orion Landing better for everyone. This program is how we say thank you.

---

## Program Overview

The Orion Landing Universal Contributor Recognition Program acknowledges developers, designers, and writers who invest time in the project. Recognition is public, lightweight, and opt-in: you decide whether to appear in the contributor list.

There are no gatekeepers and no politics. Contributions are reviewed on merit, and credit follows work.

---

## Tiers

### Contributor

**Requirement**: 1 or more merged pull requests (any type).

This is the entry point. A documentation fix counts. A one-line bug fix counts. If your PR was merged, you are a Contributor.

**Recognition**:

- Listed in `docs/community/CONTRIBUTORS.md`.
- Credited in the release notes for the version that includes your contribution.

---

### Module Author

**Requirement**: A custom module merged into the official repository or listed in the community module index.

Modules are first-class extensions of Orion Landing. Building and publishing one shows a solid understanding of the system.

**Recognition**:

- Everything in the Contributor tier.
- Listed with a "Module Author" badge in `CONTRIBUTORS.md`.
- Module listed in `docs/community/MODULE-TEMPLATE-REPO.md` (community index section).

---

### Theme Author

**Requirement**: A community theme accepted into `docs/community/THEME-GALLERY.md`.

Themes must pass the WCAG AA contrast check and the review criteria described in `THEME-GALLERY.md`.

**Recognition**:

- Everything in the Contributor tier.
- Listed with a "Theme Author" badge in `CONTRIBUTORS.md`.
- Theme credited with your name in the gallery entry.

---

### Core Contributor

**Requirement**: Significant, sustained contribution over time. This includes (but is not limited to):

- Multiple merged modules or themes.
- Major feature implementation (new admin section, new system, etc.).
- Ongoing maintenance (issue triage, review, release management).
- Substantial documentation work that meaningfully improves the project.

Core Contributors are recognized by the maintainer. There is no formal application process — the team will reach out, or you can nominate yourself by opening a discussion.

**Recognition**:

- Everything in the previous tiers.
- Listed prominently at the top of `CONTRIBUTORS.md`.
- Mentioned in the project README under "Core Contributors".
- Invited to participate in architectural discussions (GitHub Discussions / issues labeled `architecture`).

## How to Get Recognized

1. Make your contribution (PR, module, theme, docs — anything goes).
2. Once your PR is merged, open a **second PR** that adds your entry to `docs/community/CONTRIBUTORS.md`.
3. Use the format described in that file (name, GitHub, contribution, tier).
4. A maintainer will verify and merge it.

You do not need to wait for someone to add you. Self-nominating is encouraged and expected.
