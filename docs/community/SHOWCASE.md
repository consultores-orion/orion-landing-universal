## How to Add Your Site

Submissions are handled via pull request. The process takes less than five minutes.

### Requirements

- Your site must use **Orion Landing Universal v0.1.0 or later**.
- The URL must be **publicly accessible** (no password protection, no localhost).
- The site must be a real deployment, not a screenshot or mockup.

### Steps

1. **Fork** the repository on GitHub.
2. **Open** `docs/community/SHOWCASE.md` in your fork.
3. **Add a row** to the Sites table using the format below.
4. **Submit a pull request** with the title `showcase: add [Your Site Name]`.

### Row format

Copy this template and fill in every column:

```markdown
| Your Site Name | One-line description of what the site is for | Your Name or Handle | [View site](https://your-site.example.com) |
```

**Column guide**:

| Column      | What to put                                           |
| ----------- | ----------------------------------------------------- |
| Site        | The name of your site or project                      |
| Description | One sentence describing its purpose or niche          |
| Creator     | Your name, GitHub username, or company name           |
| Demo        | A markdown link using `[View site](https://your-url)` |

### What happens after you submit

A maintainer will review the PR within a few days. Review checks:

- URL is reachable and shows a landing page built with Orion Landing.
- No inappropriate or harmful content.
- Row follows the format above.

Once merged, your site appears in this table for the whole community to see.
